## [0.6.0] - TBD

Multi-target refactor. ccgate now supports both Claude Code and OpenAI Codex
CLI as PermissionRequest hook backends.

### Breaking changes

- **Top-level `ccgate init` and `ccgate metrics` are removed.** Use the
  per-target form instead:
  - `ccgate claude init`   /  `ccgate claude metrics`
  - `ccgate codex init`    /  `ccgate codex metrics`
  Running the old top-level form prints a migration message and exits with
  code `2`. The bare `ccgate` invocation (no args + stdin pipe = Claude
  Code hook) is **unchanged** and stays a permanent default; existing
  `~/.claude/settings.json` entries using `"command": "ccgate"` are not
  touched.
- **`{repo_root}/ccgate.local.jsonnet` is no longer read.** The project-local
  config path is now per-target:
  - Claude Code: `{repo_root}/.claude/ccgate.local.jsonnet`
  - Codex CLI:   `{repo_root}/.codex/ccgate.local.jsonnet`
  Move (rename) any existing root-level local config into the target
  subdirectory.

### Added

- `ccgate codex` subcommand: PermissionRequest hook for the OpenAI Codex CLI
  (`developers.openai.com/codex/hooks`). Tool-agnostic — handles Bash,
  `apply_patch`, MCP tool calls, and other surfaces Codex exposes.
- Per-target log and metrics paths under
  `$XDG_STATE_HOME/ccgate/<target>/{ccgate.log,metrics.jsonl}`
  (with read-fallback to the legacy `$XDG_STATE_HOME/ccgate/metrics.jsonl`
  so existing Claude Code users keep their full history visible through
  `ccgate claude metrics`).
- Per-target JSON schemas at `schemas/{claude,codex}.schema.json`,
  generated via `mise run schema`. Embedded defaults reference the new URLs
  for editor autocompletion.
- `.golangci.yml` plus a CI `lint` job (errcheck, errorlint, gocritic,
  gosec, govet, ineffassign, misspell, nilerr, revive, staticcheck,
  unused; gofmt + goimports formatters). Runs at zero issues.
- `mise run schema` / `mise run lint` / `mise run fmt` tasks.
- Multilingual docs: `docs/{claude,codex,configuration,architecture}.md`
  (English) plus `docs/ja/*.md` mirrors. Older `docs/README.ja.md` moved
  to `docs/ja/README.md`.
- Examples split per target: `examples/claude/ccgate.jsonnet`,
  `examples/codex/ccgate.jsonnet`.

### Changed

- `cli`/`cmd` packages refactored into the multi-target layout described in
  `docs/architecture.md`. No behavior change for the Claude Code hook.
- Schema generation now emits per-target `$id` and title strings even though
  the underlying Config struct is shared, so editors recommend the right
  schema per target and the two surfaces can diverge later without
  breaking either side's validation.

### Notes

- Codex hooks are upstream-experimental; ccgate's Codex flow is tested on
  Linux and macOS. Windows is not blocked at the binary level (the OpenAI
  Codex hooks docs list `windows_managed_dir` as a first-class field), but
  ccgate has not been exercised there yet.
- Several Codex follow-ups are intentionally out of scope for v0.6:
  `permission_mode=plan` detection, transcript JSONL parsing,
  `~/.codex/config.toml` `prefix_rules` ingestion, and
  `approval_policy` / `sandbox_mode` ingestion. They are tracked as
  follow-up issues.
